async_mqtt::client::async_publish

send PUBLISH packet

Synopsis

Declared in header <async_mqtt/asio_bind/client.hpp>

template<typename... Args>
auto
async_publish(Args&&... args);

Description

parameter detail

1st..N-1th parameters

PUBLISH packet of the Version or its constructor arguments (like std::vector::emplace_back())

  • v3_1_1::publish_packet

  • v5::publish_packet

Nth(the last) parameter

CompletionToken

  • Default Completion Token is supported

Signature

void( error_code , pubres_type )

pubres_type

  • When sending QoS0 packet, all members of pubres_type are nullopt.

  • When sending QoS1 packet, only pubres_type::puback_opt is set.

  • When sending QoS2 packet, only pubres_type::pubrec_opt and pubres_type::pubcomp are set.

error_code

  • If an error occurs during packet construction, disconnect_reason_code is set. All members of pubres_type are set to nullopt.

  • If no error occurs during packet construction, but an error occurs while checking the packet for sending, disconnect_reason_code is set. All members of pubres_type are set to nullopt.

  • If no error occurs during checking the packet for sending, but an error occurs at an underlying layer while sending a packet, underlying error is set. e.g. system, asio, beast, …​ All members of pubres_type are set to nullopt.

  • If no error occurs during the packet for sending, and if sent PUBLISH packet is QoS0, errc::success is set, if sent PUBLISH packet is QoS1 or QoS2, and corresponding response packet is received, errc::success is set if the protocol version is v3.1.1. (On v3.1.1, PUBACK, PUBREC, PUBCOMP packet has no error field.) If sent PUBLISH packet is QoS1, puback_reason_code if the protocol version is v5. If sent PUBLISH packet is QoS2, pubrec_reason_code as error if the protocol version is v5. If sent PUBLISH packet is QoS2, pubcompc_reason_code if the protocol version is v5.

Per-Operation Cancellation

This asynchronous operation supports cancellation for the following [boost::asio::cancellation_type](https://www.boost.org/doc/html/boost_asio/reference/cancellation_type.html) values:

  • cancellation_type::terminal

  • cancellation_type::partial

if they are also supported by the NextLayer type’s async_read_some and async_write_some operation.
