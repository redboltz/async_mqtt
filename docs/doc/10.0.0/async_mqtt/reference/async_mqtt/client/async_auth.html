async_mqtt::client::async_auth

send AUTH packet

Synopsis

Declared in header <async_mqtt/asio_bind/client.hpp>

template<typename... Args>
auto
async_auth(Args&&... args);

Description

parameter detail

1st..N-1th parameters

AUTH packet of the Version or its constructor arguments (like std::vector::emplace_back())

  • v5::auth_packet

Nth(the last) parameter

CompletionToken

  • Default Completion Token is supported

Signature

void( error_code )

error_code

  • If an error occurs during packet construction, and if it is a AUTH packet specific error, disconnect_reason_code is set. Otherwise, disconnect_reason_code is set. optional<connack_packet> is set to nullopt.

  • If no error occurs during packet construction, but an error occurs while checking the packet for sending, disconnect_reason_code is set. optional<connack_packet> is set to nullopt.

  • If no error occurs during checking the packet for sending, but an error occurs at an underlying layer while sending a packet, underlying error is set. e.g. system, asio, beast, …​ optional<connack_packet> is set to nullopt.

  • If no error occurs during the packet for sending, errc::success is set.

Per-Operation Cancellation

This asynchronous operation supports cancellation for the following [boost::asio::cancellation_type](https://www.boost.org/doc/html/boost_asio/reference/cancellation_type.html) values:

  • cancellation_type::terminal

  • cancellation_type::partial

if they are also supported by the NextLayer type’s async_read_some and async_write_some operation.
