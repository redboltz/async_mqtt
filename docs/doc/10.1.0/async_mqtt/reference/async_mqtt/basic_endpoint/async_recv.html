Description

Completion Token

  • Default Completion Token is supported

Signature

void( error_code , std::optional< packet_variant_type >)

error_code and packet_variant_type

  • If an error occurs at an underlying layer while receiving a packet, underlying error is set. e.g. system, asio, beast, …​ std::optional< packet_variant_type > is set to std::nullopt.

  • If there are no errors during receiving the packet, errc::success is set. std::optional< packet_variant_type > is set to basic_packet_variant .

Per-Operation Cancellation

This asynchronous operation supports cancellation for the following [boost::asio::cancellation_type](https://www.boost.org/doc/html/boost_asio/reference/cancellation_type.html) values:

  • cancellation_type::terminal

  • cancellation_type::partial

if they are also supported by the NextLayer type’s async_read_some and async_write_some operation.
